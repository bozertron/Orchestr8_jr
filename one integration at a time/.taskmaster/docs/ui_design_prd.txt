# PRD: Orchestr8 UI Design System Overhaul
**Version:** 2.0 (Design Renaissance)
**Architecture:** Marimo Reactive Dashboard
**Design Direction:** Neo-Brutalist Command Center with Cinematic Polish

## Executive Summary

Transform Orchestr8 from a functional prototype into a visually stunning command center that feels like a cross between a NASA mission control interface and a high-end developer tool. The aesthetic is "Cinematic Brutalism" - bold, uncompromising structure with refined details and dramatic visual hierarchy.

---

## 1. Design Philosophy & Aesthetic Direction

### 1.1 Core Aesthetic: "Mission Control Noir"
- **Primary Mood:** Cinematic tension of a spacecraft command center
- **Color Philosophy:** Deep space blacks with electric accent punctuation
- **Typography:** Monospace precision meets editorial elegance
- **Motion:** Purposeful micro-animations that reinforce the "live system" feeling

### 1.2 Design Principles
1. **Data Density with Clarity** - Pack information tightly but with clear visual hierarchy
2. **Ambient Feedback** - The UI should feel "alive" with subtle pulsing, glows, and state indicators
3. **Dramatic Negative Space** - Let key elements breathe while keeping secondary info compact
4. **Brutalist Structure** - Visible grid lines, exposed borders, no rounded-corner softness

- **Display/Headers:** "JetBrains Mono" or "Berkeley Mono" - technical precision

- **Fallbacks:** "SF Mono", "Consolas", monospace

### 3.2 Type Scale

## 5. Motion & Animation Specifications

### 5.1 Micro-interactions
- Button hover: Scale 1.02, shadow increase, 150ms ease
- Button active: Scale 0.98, shadow decrease, 100ms ease
- Tab switch: Content fade (200ms) with slight slide (10px)
- Badge pulse: opacity 0.7-1.0, 2s infinite for warnings
- Input focus: Border glow animation, 200ms ease

### 5.2 Page Load Sequence
1. Header fades in (0ms delay)
2. Control bar slides down (100ms delay)
3. Tabs fade in (200ms delay)
4. Content area fades in (300ms delay)

### 5.3 Data Loading States
- Skeleton loaders for table rows during scan
- Pulsing placeholder for graph during generation
- Progress bar or spinner for long operations

## 7. Technical Implementation Notes

### 7.1 Marimo CSS Integration
- Use `mo.md()` with raw HTML/CSS for custom styled components
- Leverage `mo.Html()` for complex custom elements
- Consider creating a `styles.py` cell with CSS string constants
- Use inline styles sparingly; prefer CSS classes

### 7.2 Custom CSS Injection
```python
# In a dedicated cell
def inject_styles():
    css = """
    <style>
    :root {
        --bg-void: #0a0a0f;
        /* ... all variables ... */
    }
    /* Component styles */
    </style>
    """
    return mo.Html(css)
```

### 7.3 PyVis Customization
- Use `net.set_options()` for comprehensive styling
- Custom node/edge colors via options dict
- Background and font colors in Network constructor
- Consider generating custom HTML template for PyVis
